## Data Flow

<Paragraph i18n="data-flow">
  <en>
    When a user visits a lesson, the following sequence occurs:

    1. **URL Parsing**: React Router identifies the course and lesson from the URL path (`/courses/:courseId/lessons/:lessonSlug`)
    2. **Module Resolution**: Vite's `import.meta.glob` resolves the MDX file at build time, making all lessons available as eagerly-loaded modules
    3. **State Injection**: The `LearnMDProvider` context provides global state â€” user progress, gamification points, current language, and plugin instances
    4. **MDX Rendering**: The `MDXProvider` makes interactive components (`Callout`, `Quiz`, etc.) available to the MDX content
    5. **Theme Rendering**: The `CourseViewer` component from `@learnmd/default-theme` wraps the lesson in the sidebar layout with navigation
    6. **Progress Tracking**: When a user completes a lesson or answers a quiz, progress is persisted to IndexedDB via the storage adapter
    7. **Plugin Hooks**: Registered plugins receive lifecycle events (`lesson:complete`, `progress:update`) and can trigger badge awards or other side effects
  </en>
  <es>
    Cuando un usuario visita una lecciÃ³n, ocurre la siguiente secuencia:

## Plugin System Architecture

<Paragraph i18n="plugin-arch">
  <en>
    Plugins follow a simple lifecycle model:

    1. **Registration**: Plugins are passed in the `plugins` array of `learnmd.config.ts`
    2. **Loading**: During app initialization, each plugin's `onLoad(ctx)` method is called with a `PluginContext`
    3. **Hook Registration**: Plugins use `ctx.registerHook(name, fn)` to listen for lifecycle events
    4. **Execution**: When events fire (e.g., a lesson is completed), all registered hook handlers execute
    5. **Unloading**: When the app unmounts, each plugin's optional `onUnload(ctx)` method is called for cleanup
  </en>
  <es>
    Los plugins siguen un modelo de ciclo de vida simple:

    1. **Registro**: Los plugins se pasan en el array `plugins` de `learnmd.config.ts`
    2. **Carga**: Durante la inicializaciÃ³n de la app, se llama al mÃ©todo `onLoad(ctx)` de cada plugin con un `PluginContext`
    3. **Registro de Hooks**: Los plugins usan `ctx.registerHook(name, fn)` para escuchar eventos del ciclo de vida
    4. **EjecuciÃ³n**: Cuando los eventos se disparan (ej., una lecciÃ³n se completa), todos los handlers de hooks registrados se ejecutan
    5. **Descarga**: Cuando la app se desmonta, se llama al mÃ©todo opcional `onUnload(ctx)` de cada plugin para limpieza
  </es>
