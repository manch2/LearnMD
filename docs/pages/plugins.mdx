## Plugin System Overview

<Paragraph i18n="plugin-overview">
  <en>
    Plugins are registered in your `learnmd.config.ts` file. Each plugin implements the `Plugin` interface and receives a `PluginContext` that provides access to the core engine's managers.
  </en>
  <es>
    Los plugins se registran en tu archivo `learnmd.config.ts`. Cada plugin implementa la interfaz `Plugin` y recibe un `PluginContext` que proporciona acceso a los managers del motor central.
  </es>
</Paragraph>

```typescript
// Plugin interface
interface Plugin {
  name: string;
  version: string;
  onLoad: (ctx: PluginContext) => void | Promise<void>;
  onUnload?: (ctx: PluginContext) => void | Promise<void>;
  config?: Record<string, unknown>;
}

// PluginContext available in onLoad
interface PluginContext {
  course: Course;
  storage: StorageAdapter;
  i18n: I18nAdapter;
  registerComponent: (name: string, component: unknown) => void;
  registerHook: (hook: string, fn: (...args: unknown[]) => unknown) => void;
  config: Record<string, unknown>;
}
```

<Paragraph i18n="hooks-available">
  <en>
    **Available Hooks:**

    | Hook | Description |
    |------|-------------|
    | `app:init` | Triggered when the application starts |
    | `lesson:complete` | Triggered when a user finishes a lesson |
    | `progress:update` | Triggered when course progress is updated |
  </en>
  <es>
    **Hooks Disponibles:**

    | Hook | Descripción |
    |------|-------------|
    | `app:init` | Se ejecuta cuando la aplicación inicia |
    | `lesson:complete` | Se ejecuta cuando un usuario termina una lección |
    | `progress:update` | Se ejecuta cuando el progreso del curso se actualiza |
  </es>
</Paragraph>

## Badges Plugin (`@learnmd/plugin-badges`)

<Paragraph i18n="badges-intro">
  <en>
    The Badges plugin provides gamification through achievement badges. It evaluates user progress in real-time and awards badges based on configurable criteria.
  </en>
  <es>
    El plugin de Badges proporciona gamificación a través de insignias de logros. Evalúa el progreso del usuario en tiempo real y otorga insignias basadas en criterios configurables.
  </es>
</Paragraph>

### Installation

```bash
pnpm add @learnmd/plugin-badges
```

### Configuration

<Paragraph i18n="badges-config">
  <en>
    Register the plugin in your `learnmd.config.ts`:
  </en>
  <es>
    Registra el plugin en tu `learnmd.config.ts`:
  </es>
</Paragraph>

```typescript
import { defineConfig } from '@learnmd/core';
import { BadgesPlugin } from '@learnmd/plugin-badges';

export default defineConfig({
  // ...other config
  plugins: [
    new BadgesPlugin([
      {
        id: 'first-steps',
        name: 'First Steps',
        description: 'Started your first course.',
        icon: '🚀',
        criteria: {
          type: 'course_progress',
          courseId: 'my-course',
          percentage: 1
        }
      },
      {
        id: 'halfway-there',
        name: 'Halfway There',
        description: 'Completed 50% of a course.',
        icon: '⭐',
        criteria: {
          type: 'course_progress',
          courseId: 'my-course',
          percentage: 50
        }
      },
      {
        id: 'master',
        name: 'Master',
        description: 'Completed all courses.',
        icon: '🏆',
        criteria: {
          type: 'courses_completed',
          count: 3
        }
      }
    ])
  ]
});
```

### Criteria Types

<Paragraph i18n="badges-criteria">
  <en>
    | Criteria Type | Parameters | Description |
    |---------------|-----------|-------------|
    | `course_progress` | `courseId`, `percentage` | Triggers when a specific course reaches the given completion percentage (1-100) |
    | `courses_completed` | `count` | Triggers when the user has completed the specified number of courses |
  </en>
  <es>
    | Tipo de Criterio | Parámetros | Descripción |
    |------------------|-----------|-------------|
    | `course_progress` | `courseId`, `percentage` | Se activa cuando un curso específico alcanza el porcentaje de completación dado (1-100) |
    | `courses_completed` | `count` | Se activa cuando el usuario ha completado el número especificado de cursos |
  </es>
</Paragraph>

### Badge Configuration Properties

<Paragraph i18n="badges-props">
  <en>
    | Property | Type | Required | Description |
    |----------|------|----------|-------------|
    | `id` | `string` | Yes | Unique identifier for the badge |
    | `name` | `string` | Yes | Display name |
    | `description` | `string` | Yes | What the badge represents |
    | `icon` | `string` | Yes | Emoji or icon character |
    | `criteria` | `object` | Yes | Conditions to earn the badge |
  </en>
  <es>
    | Propiedad | Tipo | Obligatorio | Descripción |
    |-----------|------|-------------|-------------|
    | `id` | `string` | Sí | Identificador único de la insignia |
    | `name` | `string` | Sí | Nombre para mostrar |
    | `description` | `string` | Sí | Qué representa la insignia |
    | `icon` | `string` | Sí | Emoji o carácter de icono |
    | `criteria` | `object` | Sí | Condiciones para ganar la insignia |
  </es>
</Paragraph>

## PDF Plugin (`@learnmd/plugin-pdf`)

<Paragraph i18n="pdf-intro">
  <en>
    The PDF plugin generates downloadable PDF certificates when users complete courses. It uses `html2canvas` and `jsPDF` to render React components to an off-screen canvas and export them as PDF documents.
  </en>
  <es>
    El plugin de PDF genera certificados PDF descargables cuando los usuarios completan cursos. Utiliza `html2canvas` y `jsPDF` para renderizar componentes React en un canvas fuera de pantalla y exportarlos como documentos PDF.
  </es>
</Paragraph>

### Installation

```bash
pnpm add @learnmd/plugin-pdf
```

### Basic Configuration

```typescript
import { defineConfig } from '@learnmd/core';
import { PDFPlugin } from '@learnmd/plugin-pdf';

export default defineConfig({
  // ...other config
  plugins: [
    new PDFPlugin()
  ]
});
```

### Custom Template

<Paragraph i18n="pdf-template">
  <en>
    You can pass a custom React component template for full control over the certificate design. The template receives the user's name, course title, completion date, and any other data you provide:
  </en>
  <es>
    Puedes pasar un componente React personalizado como plantilla para tener control total sobre el diseño del certificado. La plantilla recibe el nombre del usuario, título del curso, fecha de completación y cualquier otro dato que proporciones:
  </es>
</Paragraph>

```typescript
import { PDFPlugin } from '@learnmd/plugin-pdf';
import React from 'react';

const CertificateTemplate = ({ userName, courseTitle, date }) => (
  <div style={{
    width: '800px',
    height: '600px',
    padding: '40px',
    textAlign: 'center',
    background: 'linear-gradient(135deg, #667eea 0%, #764ba2 100%)',
    color: 'white',
    fontFamily: 'Inter, sans-serif'
  }}>
    <h1>Certificate of Completion</h1>
    <p>This certifies that</p>
    <h2>{userName}</h2>
    <p>has successfully completed</p>
    <h3>{courseTitle}</h3>
    <p>Date: {date}</p>
  </div>
);

new PDFPlugin({ template: CertificateTemplate })
```

<Callout type="tip">
  <Paragraph i18n="pdf-tip">
    <en>
      The certificate template supports all standard CSS styling, including Tailwind CSS classes if your project uses it. You can include logos, borders, and decorative elements.
    </en>
    <es>
      La plantilla del certificado soporta todo el estilo CSS estándar, incluyendo clases de Tailwind CSS si tu proyecto lo usa. Puedes incluir logos, bordes y elementos decorativos.
    </es>
  </Paragraph>
</Callout>

## Creating Your Own Plugin

<Paragraph i18n="custom-plugin">
  <en>
    You can create custom plugins to add any functionality you need. Here is a simple example of a logging plugin:
  </en>
  <es>
    Puedes crear plugins personalizados para agregar cualquier funcionalidad que necesites. Aquí hay un ejemplo simple de un plugin de logging:
  </es>
</Paragraph>

```typescript
import type { Plugin, PluginContext } from '@learnmd/core';

export class AnalyticsPlugin implements Plugin {
  name = 'analytics-plugin';
  version = '1.0.0';

  onLoad(ctx: PluginContext) {
    console.log(`Plugin "${this.name}" loaded successfully`);

    ctx.registerHook('lesson:complete', async (data) => {
      console.log(`User completed lesson: ${data.lessonSlug}`);
      // Send analytics event to your backend
    });

    ctx.registerHook('progress:update', async (data) => {
      console.log(`Progress updated for course: ${data.courseId}`);
    });
  }

  onUnload(ctx: PluginContext) {
    console.log(`Plugin "${this.name}" unloaded`);
  }
}
```

<Paragraph i18n="register-custom">
  <en>
    Then register it in your config:
  </en>
  <es>
    Luego regístralo en tu configuración:
  </es>
</Paragraph>

```typescript
import { defineConfig } from '@learnmd/core';
import { AnalyticsPlugin } from './plugins/analytics';

export default defineConfig({
  plugins: [
    new AnalyticsPlugin()
  ]
});
```
