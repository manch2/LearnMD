## Plugin Model

<Paragraph i18n="model">
  <en>
    Plugins are configured in `learnmd.config.ts`. Each plugin receives a `PluginContext` with
    access to storage, i18n, hooks, and component registration. The theme owns layout and visual
    styling, while plugins provide behavior and optional UI blocks that render inside stable slots.
  </en>
  <es>
    Los plugins se configuran en `learnmd.config.ts`. Cada plugin recibe un `PluginContext` con
    acceso a storage, i18n, hooks y registro de componentes. El theme es el dueno del layout y
    del estilo visual, mientras que los plugins aportan comportamiento y bloques UI opcionales
    que se renderizan dentro de slots estables.
  </es>
</Paragraph>

```typescript
import type {
  Plugin,
  PluginContext,
  PluginSlotName,
} from '@learnmd/core';

interface Plugin {
  name: string;
  version: string;
  onLoad: (ctx: PluginContext) => void | Promise<void>;
  onUnload?: (ctx: PluginContext) => void | Promise<void>;
}

type PluginSlotName =
  | 'profile:summary'
  | 'profile:courseActions'
  | 'profile:sections';

interface PluginContext {
  storage: LearnMDStorage;
  i18n: I18nAdapter;
  registerHook: (hook: string, fn: (...args: unknown[]) => unknown) => void;
  registerComponent: (
    registration:
      | string
      | {
          slot: PluginSlotName;
          name: string;
          component: unknown;
          order?: number;
          metadata?: Record<string, unknown>;
        },
    component?: unknown
  ) => void;
  config: Record<string, unknown>;
}
```

<Paragraph i18n="hooks">
  <en>
    Common hooks in beta.3 include `app:init`, `lesson:complete`, `quiz:submit`, `progress:update`
    and `profile:update`. Use hooks for behavior, and slots only when your plugin needs UI.
  </en>
  <es>
    Los hooks comunes en beta.3 incluyen `app:init`, `lesson:complete`, `quiz:submit`,
    `progress:update` y `profile:update`. Usa hooks para comportamiento, y slots solo cuando tu
    plugin necesite UI.
  </es>
</Paragraph>

## Official Plugins

### Badges Plugin (`@learnmd/plugin-badges`)

<Paragraph i18n="badges">
  <en>
    The badges plugin is responsible for awarding insignias. It consumes the public profile state,
    including `globalScore`, and evaluates badge criteria without taking ownership of point
    calculation.
  </en>
  <es>
    El plugin de badges es responsable de otorgar insignias. Consume el estado publico del perfil,
    incluido `globalScore`, y evalua criterios de badges sin aduenarse del calculo de puntos.
  </es>
</Paragraph>

```typescript
import { defineConfig } from '@learnmd/core';
import { BadgesPlugin } from '@learnmd/plugin-badges';

export default defineConfig({
  gamification: {
    points: {
      lessonCompletion: 100,
      quizPassed: 10,
      quizPerfectScore: 15,
    },
  },
  plugins: [
    new BadgesPlugin([
      {
        id: 'first-steps',
        name: 'First Steps',
        description: 'Complete your first lesson.',
        icon: '??',
        criteria: {
          type: 'course_progress',
          courseId: 'getting-started',
          percentage: 1,
        },
      },
      {
        id: 'score-500',
        name: 'Momentum',
        description: 'Reach 500 global points.',
        icon: '??',
        criteria: {
          type: 'global_score',
          score: 500,
        },
      },
    ]),
  ],
});
```

<Paragraph i18n="badges-criteria">
  <en>
    Supported criteria include `course_progress`, `courses_completed`, and `global_score`. This
    lets you build badges from progress, completions, or the public global score generated by the
    gamification system.
  </en>
  <es>
    Los criterios soportados incluyen `course_progress`, `courses_completed` y `global_score`.
    Esto te permite construir badges desde progreso, cursos completados o el puntaje global
    publico generado por el sistema de gamificacion.
  </es>
</Paragraph>

### PDF Plugin (`@learnmd/plugin-pdf`)

<Paragraph i18n="pdf">
  <en>
    The PDF plugin is the only owner of certificate downloads. If you do not register
    `new PDFPlugin()`, LearnMD will not render certificate actions in the profile and users will
    not be able to generate PDFs from the UI.
  </en>
  <es>
    El plugin PDF es el unico dueno de las descargas de certificados. Si no registras
    `new PDFPlugin()`, LearnMD no renderiza acciones de certificados en el perfil y los usuarios no
    podran generar PDFs desde la UI.
  </es>
</Paragraph>

```typescript
import { defineConfig } from '@learnmd/core';
import { PDFPlugin } from '@learnmd/plugin-pdf';

export default defineConfig({
  plugins: [
    new PDFPlugin({
      signature: 'LearnMD Team',
    }),
  ],
});
```

<Paragraph i18n="pdf-ui">
  <en>
    In beta.3, the plugin injects its UI through the `profile:courseActions` slot. The default
    theme decides where that slot is shown, but the action only exists when the plugin is present.
  </en>
  <es>
    En beta.3, el plugin inyecta su UI a traves del slot `profile:courseActions`. El theme por
    defecto decide donde se muestra ese slot, pero la accion solo existe cuando el plugin esta
    presente.
  </es>
</Paragraph>

```typescript
import React from 'react';
import { PDFPlugin } from '@learnmd/plugin-pdf';

const CertificateTemplate = (
  <div
    style={{
      width: '11in',
      height: '8.5in',
      padding: '48px',
      background: '#ffffff',
      color: '#0f172a',
      fontFamily: 'Georgia, serif',
    }}
  >
    <h1>Certificate of Completion</h1>
    <p>This certifies successful completion.</p>
  </div>
);

new PDFPlugin({
  signature: 'LearnMD Instructors',
  template: CertificateTemplate,
});
```

## UI Slots

<Paragraph i18n="slots">
  <en>
    Plugins with UI should target theme slots instead of patching the theme. This keeps LearnMD
    modular: the theme controls layout and styling primitives, while plugins register React
    components for known integration points.
  </en>
  <es>
    Los plugins con UI deben apuntar a slots del theme en lugar de parchear el theme. Esto mantiene
    LearnMD modular: el theme controla layout y primitivas visuales, mientras los plugins registran
    componentes React para puntos de integracion conocidos.
  </es>
</Paragraph>

<Paragraph i18n="slots-list">
  <en>
    Current public slots:

    | Slot | Purpose |
    |------|---------|
    | `profile:summary` | Small summary widgets near the top of the profile |
    | `profile:courseActions` | Per-course actions such as certificates |
    | `profile:sections` | Additional profile sections rendered below badges |
  </en>
  <es>
    Slots publicos actuales:

    | Slot | Proposito |
    |------|-----------|
    | `profile:summary` | Widgets pequenos cerca de la parte superior del perfil |
    | `profile:courseActions` | Acciones por curso, como certificados |
    | `profile:sections` | Secciones adicionales del perfil debajo de badges |
  </es>
</Paragraph>

```typescript
import React from 'react';
import { BasePlugin, type PluginContext } from '@learnmd/core';

function WelcomePanel() {
  return (
    <div className="rounded-xl border border-[rgb(var(--border-color))] bg-[rgb(var(--bg-primary))] p-4">
      Custom plugin UI rendered inside the theme.
    </div>
  );
}

export class WelcomePlugin extends BasePlugin {
  constructor() {
    super('welcome', '1.0.0');
  }

  onLoad(ctx: PluginContext) {
    ctx.registerComponent({
      slot: 'profile:sections',
      name: 'welcome-panel',
      order: 10,
      component: WelcomePanel,
    });
  }
}
```

<Callout type="tip">
  <Paragraph i18n="slots-tip">
    <en>
      A plugin can ship its own UI logic, but it should reuse the theme's CSS variables and visual
      tokens. That way users can change colors, spacing, and typography from `theme` without
      rewriting each plugin.
    </en>
    <es>
      Un plugin puede traer su propia logica UI, pero debe reutilizar las variables CSS y tokens
      visuales del theme. Asi los usuarios pueden cambiar colores, espaciado y tipografia desde
      `theme` sin reescribir cada plugin.
    </es>
  </Paragraph>
</Callout>

## Creating Your Own Plugin

```typescript
import { BasePlugin, type PluginContext } from '@learnmd/core';

export class AnalyticsPlugin extends BasePlugin {
  constructor() {
    super('analytics', '1.0.0');
  }

  onLoad(ctx: PluginContext) {
    ctx.registerHook('lesson:complete', async (payload) => {
      console.log('Lesson completed', payload);
    });

    ctx.registerHook('progress:update', async (payload) => {
      console.log('Progress updated', payload);
    });
  }
}
```

<Paragraph i18n="custom-register">
  <en>
    Register custom plugins in `plugins`. If your plugin needs UI, register slot components in
    `onLoad`; if it is behavior-only, hooks are enough.
  </en>
  <es>
    Registra plugins personalizados en `plugins`. Si tu plugin necesita UI, registra componentes
    de slot en `onLoad`; si es solo de comportamiento, los hooks son suficientes.
  </es>
</Paragraph>

```typescript
import { defineConfig } from '@learnmd/core';
import { AnalyticsPlugin } from './plugins/analytics';
import { WelcomePlugin } from './plugins/welcome';

export default defineConfig({
  plugins: [new AnalyticsPlugin(), new WelcomePlugin()],
});
```
