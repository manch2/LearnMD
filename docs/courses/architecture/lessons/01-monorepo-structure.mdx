  <es>Estructura del monorepo</es>

<Paragraph i18n="intro">
  <en>
    LearnMD uses a monorepo to keep packages, docs, plugins, and examples aligned. Turborepo coordinates builds and tests so changes in the core, theme, and CLI can evolve together.
  </en>
  <es>
    LearnMD usa un monorepo para mantener alineados paquetes, documentación, plugins y ejemplos. Turborepo coordina builds y pruebas para que el core, el theme y el CLI evolucionen juntos.
  </es>
</Paragraph>

## Benefits / Beneficios

<Paragraph i18n="benefits">
  <en>
    A monorepo makes code sharing, dependency management, and release coordination much easier. Shared types and components move together instead of drifting across separate repositories.
  </en>
  <es>
    Un monorepo hace mucho más fácil compartir código, manejar dependencias y coordinar releases. Los tipos y componentes compartidos avanzan juntos en lugar de desviarse entre repositorios separados.
  </es>
</Paragraph>

## Key Directories / Directorios clave

<Paragraph i18n="directories">
  <en>
    `packages/` contains the framework packages, `docs/` is a real LearnMD site used as documentation, and `examples/` contains showcase projects generated from the CLI and then customized with content.
  </en>
  <es>
    `packages/` contiene los paquetes del framework, `docs/` es un sitio real de LearnMD usado como documentación y `examples/` contiene proyectos de demostración generados desde el CLI y luego personalizados con contenido.
  </es>
</Paragraph>

<Callout type="info">
  <Paragraph i18n="contributing">

    </en>
    <es>
      Si quieres contribuir a LearnMD, empieza entendiendo la estructura del workspace antes de modificar un paquete de forma aislada.
    </es>
  </Paragraph>
</Callout>
