# Core Engine

The `@learnmd/core` package is the heart of the framework. It handles the logic for parsing, routing, and state management.

## Key Components

### 1. Parser

The Parser takes your MDX files and extracts:
-   **Frontmatter**: Metadata like title, description, and tags.
-   **Content**: The rendered React components and markdown.
-   **Structure**: Identifying headings for the table of contents.

### 2. Router Manager

Manages the mapping between URLs and your content:
-   `/courses/:courseId`: Course overview.
-   `/courses/:courseId/lessons/:lessonSlug`: Individual lessons.
-   `/:pagePath`: Custom pages.

### 3. LearnMD Provider

A React Context provider that makes the global state available to your entire application:
-   `progress`: Current user progress in all courses.
-   `gamification`: Points, badges, and streaks.
-   `i18n`: Translations and language settings.

## Extensibility via Hooks

The core engine provides a robust hook system that plugins can use to tap into the application lifecycle:

```typescript
// Example plugin hook
export const MyPlugin = {
  name: 'my-plugin',
  setup: (ctx) => {
    ctx.registerHook('lesson:complete', async (data) => {
      console.log(`Lesson ${data.lessonSlug} completed!`);
    });
  }
};
```

<Callout type="warning">
  Avoid modifying the core engine directly unless you are building a feature that benefits all users. Use plugins instead!
</Callout>
