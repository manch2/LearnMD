<Title i18n="title">
  <en>Customizing the Theme</en>
  <es>Personalizar el theme</es>
</Title>

<Paragraph i18n="intro">
  <en>
    LearnMD ships with a default theme, but its appearance is intentionally configurable. Brand colors, fonts, width, and dark mode tokens are controlled through `learnmd.config.ts`.
  </en>
  <es>
    LearnMD incluye un theme por defecto, pero su apariencia es intencionalmente configurable. Los colores de marca, las fuentes, el ancho y los tokens de modo oscuro se controlan desde `learnmd.config.ts`.
  </es>
</Paragraph>

```typescript
theme: {
  primaryColor: '#8b5cf6',
  secondaryColor: '#0f766e',
  headingFontFamily: '"Space Grotesk", "Inter", system-ui, sans-serif',
  contentMaxWidth: '72rem',
  darkMode: {
    enabled: true,
  },
}
```

<Paragraph i18n="theme-fields">
  <en>
    Start with configuration before forking styles. In many cases `primaryColor`, `secondaryColor`, `fontFamily`, `headingFontFamily`, and `contentMaxWidth` already provide enough freedom.
  </en>
  <es>
    Empieza por la configuración antes de bifurcar estilos. En muchos casos `primaryColor`, `secondaryColor`, `fontFamily`, `headingFontFamily` y `contentMaxWidth` ya ofrecen suficiente libertad.
  </es>
</Paragraph>

<Callout type="info">
  <Paragraph i18n="info">
    <en>
      Plugins with UI should reuse theme CSS variables so they inherit the project look automatically.
    </en>
    <es>
      Los plugins con UI deben reutilizar las variables CSS del theme para heredar automáticamente la apariencia del proyecto.
    </es>
  </Paragraph>
</Callout>
