# Plugin Lifecycle

LearnMD is highly extensible via its plugin system. Plugins allow you to tap into various stages of the application's lifecycle and add custom functionality.

## Core Plugin API

A plugin is simply an object with a `name` and a `setup` function:

```typescript
export interface Plugin {
  name: string;
  setup: (ctx: PluginContext) => void | Promise<void>;
}
```

The `setup` function receives a `PluginContext` that provides access to the core engine's managers (i18n, storage, gamification, router) and a `registerHook` function.

## Available Hooks

The core engine provides several hooks that plugins can use:

-   `app:init`: Triggered when the application starts.
-   `lesson:complete`: Triggered when a user finishes a lesson.
-   `progress:update`: Triggered when course progress is updated.

## Creating a Simple Plugin

```typescript
export const LoggerPlugin = {
  name: 'logger-plugin',
  setup: (ctx) => {
    ctx.registerHook('lesson:complete', async (data) => {
      console.log(`User finished lesson: ${data.lessonSlug}`);
    });
  }
};
```

<Callout type="info">
  Plugins are registered in the `learnmd.config.ts` file under the `plugins` array.
</Callout>
