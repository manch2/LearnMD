<Paragraph i18n="intro">
  <en>
    The PDF plugin owns certificate generation in LearnMD `0.0.4-beta.1`. If the plugin is not registered, the default UI must not show certificate actions.
  </en>
  <es>
    El plugin PDF es dueño de la generación de certificados en LearnMD `0.0.4-beta.1`. Si el plugin no está registrado, la UI por defecto no debe mostrar acciones de certificados.
  </es>
</Paragraph>

## Registration / Registro

```typescript
import { defineConfig } from '@learnmd/core';
import { PDFPlugin } from '@learnmd/plugin-pdf';

export default defineConfig({
  plugins: [
    new PDFPlugin({
      signature: 'LearnMD Team',
    }),
  ],
});
```

<Paragraph i18n="behavior">
  <en>
    The plugin registers a UI action for completed courses in the `profile:courseActions` slot. Without the plugin, users do not have an alternate certificate flow in the default interface.
  </en>
  <es>
    El plugin registra una acción de UI para cursos completados en el slot `profile:courseActions`. Sin el plugin, los usuarios no tienen un flujo alternativo de certificados en la interfaz por defecto.
  </es>
</Paragraph>

## Course Templates & Custom Templates

```typescript
import React from 'react';
import { PDFPlugin, CertificateTemplateProps } from '@learnmd/plugin-pdf';

const CustomCertificate: React.FC<CertificateTemplateProps> = ({ 
  courseTitle, 
  studentName, 
  completionDate 
}) => (
  <div style={{ width: '11in', height: '8.5in', padding: '48px', backgroundColor: 'white' }}>
    <h1>Certificate of Completion</h1>
    <h2>{studentName}</h2>
    <p>Completed: {courseTitle} on {completionDate.toLocaleDateString()}</p>
  </div>
);

new PDFPlugin({
  signature: 'Lead Instructor',
  defaultTemplate: 'classic', // Built-in templates: 'classic' or 'modern'
  courseTemplates: {
    'onboarding-course': 'modern',
    'special-course': CustomCertificate
  }
});
```

<Callout type="warning">
  <Paragraph i18n="warning">
    <en>
      Certificate UI should always be gated by plugin registration. Do not import certificate generation directly from the theme layer.
    </en>
    <es>
      La UI de certificados siempre debe estar controlada por el registro del plugin. No importes la generación de certificados directamente desde la capa del theme.
    </es>
  </Paragraph>
</Callout>
